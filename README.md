## 9. Engineering Methodology
Typical execution model:
1. define constraints and failure boundaries
2. design architecture and interfaces
3. implement in small, verifiable increments
4. benchmark and profile critical paths
5. validate via tests and reproducible checks
6. instrument for observability and operational feedback
7. iterate using measured production behavior

## 10. Security & Reliability Philosophy
Security and reliability are baseline requirements, not post-implementation phases. Systems should assume adversarial conditions, partial failure, and operational drift. Engineering decisions are evaluated against blast radius, recovery characteristics, and maintainability under incident pressure.
